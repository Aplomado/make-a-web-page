Lesson 1: The Basics

How the Internet Works

The Internet is the interconnection of millions of computers. This is one giant computer network. Some are consuming content using Web Browsers or Email Clients. Some computers are holding those files, and serving them out when they are needed. These are appropriately called Servers.
When a web browser goes to a url, it is requesting the file(s) for a web site to be sent via HTTP from a server. HTTP is the communication protocol the Internet works on. The server finds the needed file(s), then send them back so the web browser can interpret them so they make sense to us lowly humans.

HTML

HTML stands for Hypertext Markup Language. HTML pages form the vast majority of all content on the Internet.

Tags and Elements

The parts of an HTML pages, other than the text, is called Elements. This is the combination of Tags, and other things like values and URL's. Tags are used to tell the web browser the type of Element that is being seen.

Inline and Block Elements

Elements in HTML are either Inline or Block. Inline elements are used inside of the text itself. Block elements form a box or block around the content inside of them.

Lesson 2: Structured Documents

HTML and CSS

HTML is really the structure of a webpage. It forms the walls and floor of the website if it were a house. Any thing constructed has to have a basic structure.
CSS applies the styling to the webpage. It can control font type, font size, even color. CSS is the artistic decor of the HTML house.

Ticky Tacky Boxes

All HTML elements of a webpage either Inline or Block. Block elements are all boxes. It may not seem so with so many circles, triangles, and squiggles. All of the areas set aside for the shapes to exist are boxes. This also includes the areas that contain text.

Web Browser Dev Tools

The Dev Tools in most web browsers are good ways to inspect and test changes to webpages. Testing changes on webpages is temporary. It does not save these changes to the HTML or CSS files. A simple refresh of the page from the original file is enough to reset the page back to its original state.

Development Text Editor

It is important to use a text editor that is designed for software development. There are several good editors avalible. Because I use a Macintosh, I prefer some editors that may or may not be availible to other platforms. This Nanodegree from Udacity uses the Sublime Text editor as the default. I also have used Coderunner 2, and Atom for the Python class I have taken.

Leasson 3: More CSS, Inline Style, Checking Code, and Goundhog Day

More CSS

As mentioned before, CSS file work in conjunction with HTML documents to style the webpage. To do this the CSS needs to be able to deferenciate between the diferent section of the page. One way of doing this is to encapsulate parts of the code in containers like Div's. Div tags can appear inside of other Div's, and they can be nested as far as needed.
In order to be able to call these Div's in the CSS, they need to have Classes asigned to them. A Class is a tag for the container adds that container to a group so the same styling can be added to all of the group. This allows for quicker coding than doing all of the styling indiviually.
There is one other tag that can be used. The ID tag allows someone to spicificlly identify one single container in the code. ID's are used to lable one indiviual element for special asignment in CSS. It is not recomended doing this unless needed because by calling one element out as spiecal, it adds more code, which is more time and energy.

Inline Style

Style can be added to the HTML directly by useing the style tag. By encapulating CSS code in Style tags in the HTML, a CSS file may not be needed for a small project. This is not recomended, but for small projects, this may be excepable.

Checking Code

Even though we do our best at not making mistakes in the code we write, mistakes do happen. Sometimes we are on a role, in the zone, or whatever term you would like to use, and we forget to close a tag, or forget a Div. If after finishing the section that was being worked on things do not look right, try to isolate the postion of the code that is the offending party, then work in from there.
Also, there is a website that can check for validation of the code. HTML can be checked here: validator.w3.org. CSS can be checked here: jigsaw.w3.org.

Groundhog Day

It is important to minimise repetition whenever possible. By reducing repetition we save time and effort. We also minimize mistakes by writing less code. Also, by keeping the code smaller, we inherently keep file size smaller which is always helpful economiclly, and for loading speads.
